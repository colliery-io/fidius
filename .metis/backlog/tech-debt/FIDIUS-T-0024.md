## Objective

The `#[plugin_impl]` macro currently hardcodes `capabilities: 0u64` in the generated descriptor. It should compute the bitfield by checking which optional methods the impl block provides, using the `{Trait}_OPTIONAL_METHODS` const and `{Trait}_CAP_{METHOD}` constants generated by `#[plugin_interface]`.

## Technical Debt Impact

- **Current Problems**: `PluginHandle::has_capability()` always returns false, even when the optional method is implemented and callable. Host code can't introspect optional method support.
- **Benefits of Fixing**: Host can check capability bits before calling optional methods, matching the spec's evolution model.
- **Risk Assessment**: Low risk — the methods already work, this just fixes the metadata.

## Implementation Notes

### Technical Approach

File: `fidius-macro/src/impl_macro.rs`, `generate_descriptor()` function.

The `{Trait}_OPTIONAL_METHODS` const (generated by `#[plugin_interface]`) lists optional method names. For each method in the impl block that appears in `OPTIONAL_METHODS`, OR the corresponding `{Trait}_CAP_{METHOD}` bit into the capabilities value.

Since the proc macro can't read constants from other crates at expansion time, generate code that computes the bitfield at compile time using the constants.
