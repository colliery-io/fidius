# Add crate_path to InterfaceAttrs and plugin_interface codegen

## Parent Initiative

[[FIDIUS-I-0011]]

## Objective

Parse an optional `crate = "..."` attribute in `#[plugin_interface]`, store it as `crate_path` in `InterfaceAttrs`, and use it in all code generated by the interface macro. Embed the path in the companion module so `plugin_impl` can read it.

## Acceptance Criteria

## Acceptance Criteria

## Acceptance Criteria

- [ ] `InterfaceAttrs` gains `crate_path: syn::Path` field, defaulting to `fidius` when absent
- [ ] `#[plugin_interface(version = 1, buffer = PluginAllocated, crate = "my_crate::fidius")]` parses correctly
- [ ] All `fidius::` paths in `interface.rs` codegen use `#crate_path` instead
- [ ] Companion module (`__fidius_{TraitName}`) exports the crate path as a type alias or const so `plugin_impl` can read it
- [ ] Existing code without `crate = "..."` continues to work (defaults to `fidius`)
- [ ] IR unit tests updated for crate_path parsing

## Implementation Notes

### Files to modify
- `fidius-macro/src/ir.rs` — add `crate_path` field to `InterfaceAttrs`, parse `crate = "..."` in `Parse` impl
- `fidius-macro/src/interface.rs` — replace all `quote!(fidius::...)` with `quote!(#crate_path::...)`, embed path in companion module

### Embedding approach
The companion module should export something like:
```rust
pub mod __fidius_MyTrait {
    pub use my_crate::fidius as __fidius_crate;
    // ... existing vtable, descriptor builder, etc.
}
```
Then `plugin_impl` can reference `__fidius_MyTrait::__fidius_crate::wire::serialize(...)`.
